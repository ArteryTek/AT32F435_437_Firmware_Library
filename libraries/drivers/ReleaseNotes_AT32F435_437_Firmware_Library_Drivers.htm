Release Notes for

AT32F435_437 Firmware Library Drivers

V2.0.7-2025/11/24

1.         ADC

l  新增adc_combine_mode_set()函数，设置主从组合模式。

2.         CRM

l  修正bit位域名adc14rst为tmr14rst，新增adc2en bit位域名。

3.         GPIO

l  新增gpio_bits_toggle()函数。

4.         QSPI

l  修正xip_cmd_w2_bit.xipr_dcnt bit宽度为6 bit。

5.         TMR

l  更新tmr_pwm_input_config()、tmr_encoder_mode_config()和tmr_output_channel_config()函数。通道模式配置前关闭通道使能，否则通道模式无法切换。

6.         更新部分注释。

 

V2.0.6-2024/12/13

1.         PWC

l  调整LDO输出档位设定，移除LDO配置项PWC_LDO_OUTPUT_1V0。

2.         CAN

l  更新发送中止函数can_transmit_cancel()写法，以避免误操作而导致中止命令在各邮箱间互相影响。

3.         USB

l  更新host模式下usb_hch_halt()函数，完善通道halt处理流程。

l  更新host模式下usb_hc_enable()函数，使能对应通道时确保清除其原有状态。

l  更新host模式下usb_host_disable()函数，更新关闭通道流程，以避免误打开其余通道。

l  更新device模式下usb_ept_open()函数，增加同步传输类型的处理，忽略其帧号。

l  更新device模式下usb_ept_close()函数，完善关闭端点流程。

 

V2.0.5-2024/08/08

1.         QSPI

l  新增qspi_auto_ispc_enable()函数，用于总线输入时钟的相位匹配。

l  修正fifo深度定义名，将QSPI_DMA_FIFO_THOD_WORD32改为QSPI_DMA_FIFO_THOD_WORD24。

l  更新qspi_flag_clear()函数的写法，修正AC6编译警告。

 

V2.0.4-2024/01/05

1.         完善部分驱动流程在IAR7.4下的编译警告。

2.         完善EMAC的PTP驱动。

3.         更新dvp_flag_get()和dvp_interrupt_flag_get()函数。

 

V2.0.3-2023/10/26

1.         各IP驱动新增interrupt_flag_get()函数。

2.         优化系统时钟切换流程。

3.         更新IRTMR驱动。

4.         更新ADC驱动，完善软触发流程。

5.         更新部分注释。

 

V2.0.2-2023/08/04

1.         优化了qspi_xip_enable()函数流程。

2.         更新了flash_slib_enable()函数中sector条件判断流程。

3.         CRC驱动新增多项式及多项式宽度可修改的支持。

4.         更新了adc_ordinary_channel_set()和adc_preempt_channel_set()函数流程。

5.         更新部分注释。

 

V2.0.1-2023/02/16

1.         调整CRM emac lowpower枚举定义位置为AT32F437系列。

 

V2.0.0-2022/11/18

1.         更新了xmc_ext_timing_config()函数。

2.         更新当HICK作为PLL时钟源时CRM_HICK48_NODIV作为默认设置。
